## 1. 快速部署（一键启动）

### 1.1 系统要求

| 依赖 | 版本 | 必需 |
|------|------|------|
| Node.js | 20+ | ✓ |
| Python | 3.11+ | ✓ |
| Git | 任意 | ✓ |
| Ollama | 最新 | 可选（本地AI） |
| Docker | 最新 | 可选 |

### 1.2 Windows部署

```bash
# 1. 克隆项目
git clone <repository-url>
cd smart-agriculture

# 2. 运行一键启动脚本
setup.bat
```

### 1.3 Linux/macOS部署

```bash
# 1. 克隆项目
git clone <repository-url>
cd smart-agriculture

# 2. 设置执行权限
chmod +x setup.sh

# 3. 运行一键启动脚本
./setup.sh
```

### 1.4 启动流程说明

一键启动脚本会自动执行以下步骤：

```
[1/6] 检测Node.js
[2/6] 检测Python
[3/6] 安装Node.js依赖（npm install）
[4/6] 安装Python依赖（创建虚拟环境 + pip install）
[5/6] 初始化数据库（SQLite）
[6/6] 创建环境变量文件（.env.local）
      ↓
启动AI服务（Ollama + YOLO + RAG）
      ↓
启动Web服务（http://localhost:3000）
```

## 2. 手动部署

### 2.1 安装依赖

**Node.js依赖：**
```bash
npm install
```

**Python依赖：**
```bash
cd inference-service
python -m venv venv

# Windows
venv\Scripts\activate
pip install -r requirements.txt
pip install -r requirements-rag.txt

# Linux/macOS
source venv/bin/activate
pip install -r requirements.txt
pip install -r requirements-rag.txt
```

### 2.2 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env.local

# 编辑环境变量
# Windows: notepad .env.local
# Linux/macOS: nano .env.local
```

**环境变量说明：**

```env
# 数据库配置（默认SQLite）
DATABASE_TYPE=sqlite
SQLITE_DB_PATH=./smart_agriculture.db

# AI模式：local / network / auto
AI_MODE=auto

# 本地AI服务
OLLAMA_HOST=http://localhost:11434
OLLAMA_MODEL=qwen3:1.7b-q4_K_M
INFERENCE_HOST=http://localhost:5000
RAG_SERVICE_URL=http://localhost:5001

# 网络API（备选）
AI_PROVIDER=openai
OPENAI_API_KEY=sk-your-key-here
OPENAI_API_BASE=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o-mini

# 应用配置
PORT=3000
NODE_ENV=development
```

### 2.3 初始化数据库

```bash
node scripts/init-db.js
```

### 2.4 启动服务

**启动AI服务：**
```bash
# 终端1：启动Ollama
ollama serve

# 终端2：启动YOLO推理服务
cd inference-service
source venv/bin/activate  # Windows: venv\Scripts\activate
python app.py

# 终端3：启动RAG服务
python rag_app.py
```

**启动Web服务：**
```bash
npm run dev
```

## 4. AI服务配置

### 4.1 本地AI（推荐）

本地AI服务提供最快的响应速度和隐私保护。

**安装AI服务：**

```bash
# Windows
powershell -ExecutionPolicy Bypass -File scripts/setup-ai.ps1

# Linux/macOS
chmod +x scripts/setup-ai.sh
./scripts/setup-ai.sh
```

**手动安装：**

```bash
# 1. 安装Ollama
# Windows: 访问 https://ollama.ai/ 下载安装
# Linux/macOS:
curl -fsSL https://ollama.ai/install.sh | sh

# 2. 拉取模型
ollama pull qwen3:1.7b-q4_K_M

# 3. 安装Python依赖
cd inference-service
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
pip install -r requirements-rag.txt
```

**配置环境变量：**

```env
AI_MODE=local
OLLAMA_HOST=http://localhost:11434
OLLAMA_MODEL=qwen3:1.7b-q4_K_M
INFERENCE_HOST=http://localhost:5000
RAG_SERVICE_URL=http://localhost:5001
```

### 4.2 网络API（备选）

当本地AI服务不可用时，可使用网络API。

**支持的API提供商：**

| 提供商 | 环境变量 | 说明 |
|--------|----------|------|
| OpenAI | AI_PROVIDER=openai | GPT-4o-mini等 |
| 通义千问 | AI_PROVIDER=tongyi | qwen-turbo等 |
| 智谱AI | AI_PROVIDER=zhipu | glm-4-flash等 |
| 自定义 | AI_PROVIDER=custom | 任意OpenAI兼容API |

**OpenAI配置：**

```env

AI_PROVIDER=openai
OPENAI_API_KEY=sk-your-key-here
OPENAI_API_BASE=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o-mini
```

**通义千问配置：**

```env
AI_MODE=network
AI_PROVIDER=tongyi
DASHSCOPE_API_KEY=sk-your-key-here
DASHSCOPE_MODEL=qwen-turbo
```

**智谱AI配置：**

```env

AI_PROVIDER=zhipu

```

**自定义API配置：**

```env

CUSTOM_API_MODEL=your-model
```

## 5. 生产环境部署

### 5.1 使用PM2管理进程

```bash
# 安装PM2
npm install -g pm2

# 构建项目
npm run build

# 启动应用
pm2 start npm --name "smart-agriculture" -- start

# 设置开机自启
pm2 startup
pm2 save

# 查看状态
pm2 status

# 查看日志
pm2 logs smart-agriculture
```

### 5.2 使用Nginx反向代理

**安装Nginx：**

```bash
# Ubuntu/Debian
sudo apt-get install nginx

# CentOS/RHEL
sudo yum install nginx
```

**配置Nginx：**

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # WebSocket支持
    location /api/websocket {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400;
    }
}
```

**启用配置：**

```bash
sudo ln -s /etc/nginx/sites-available/smart-agriculture /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 5.3 配置HTTPS

```bash
# 安装Certbot
sudo apt-get install certbot python3-certbot-nginx

# 获取SSL证书
sudo certbot --nginx -d your-domain.com

# 验证自动续期
sudo certbot renew --dry-run
```

## 6. 常见问题

### 6.1 依赖检测失败

**运行依赖检测脚本：**

```bash
# Windows
powershell -ExecutionPolicy Bypass -File scripts/check-deps.ps1

# Linux/macOS
chmod +x scripts/check-deps.sh
./scripts/check-deps.sh
```

### 6.2 npm install失败

```bash
# 清除缓存
npm cache clean --force

# 删除node_modules重新安装
rm -rf node_modules package-lock.json
npm install
```

### 6.3 Python依赖安装失败

```bash
# 升级pip
python -m pip install --upgrade pip

# 使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 6.4 Ollama安装失败

**Windows：**
- 访问 https://ollama.ai/ 下载安装包

**Linux：**
```bash
curl -fsSL https://ollama.ai/install.sh | sh
```

**macOS：**
```bash
brew install ollama
```

### 6.5 数据库连接失败

默认使用SQLite，无需额外配置。如需使用MySQL：

```env
DATABASE_TYPE=mysql
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=smart_agriculture
```

### 6.6 端口被占用

```bash
# 查看端口占用
# Windows
netstat -ano | findstr :3000

# Linux/macOS
lsof -i :3000

# 杀死进程
# Windows
taskkill /PID <PID> /F

# Linux/macOS
kill -9 <PID>
```

### 6.7 Windows执行策略被禁止

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## 7. 项目功能说明

### 7.1 核心功能

- **传感器监控**：实时监控温度、湿度、光照、土壤湿度等传感器数据
- **执行器控制**：远程控制水泵、风扇、补光灯等设备
- **AI智能诊断**：基于传感器数据的智能分析和建议
- **知识库管理**：农业知识库的CRUD和语义搜索
- **报警系统**：阈值配置和报警记录
- **多地点管理**：支持多个农场/区域的管理

### 7.2 API接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/sensors | GET | 获取传感器列表 |
| /api/sensors/[id]/data | GET/POST | 传感器数据读写 |
| /api/actuators | GET | 获取执行器列表 |
| /api/actuators/[id]/commands | GET/POST | 执行器控制 |
| /api/knowledge | GET/POST | 知识库CRUD |
| /api/knowledge/search | POST | 知识库搜索 |
| /api/ai/diagnosis | POST | AI诊断 |
| /api/device/report | POST | 设备数据上报 |

### 7.3 前端页面

| 页面 | 路径 | 说明 |
|------|------|------|
| 数据概览 | / | 仪表板，展示所有传感器概览 |
| 精细数据 | /detailed | 传感器详细数据表格 |
| 数据分析 | /analysis | 数据图表分析 |
| 设备连接 | /devices | 设备管理 |
| 执行器控制 | /actuators | 执行器远程控制 |
| AI监测 | /ai-monitor | AI智能诊断 |
| 知识库 | /knowledge | 知识库管理 |
