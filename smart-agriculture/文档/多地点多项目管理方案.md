# 智慧农业物联网平台 - 多地点多项目管理方案

## 更新日期：2026-06-23

---

## 1. 业务场景分析

### 1.1 典型用户场景

| 场景 | 描述 | 设备规模 |
|------|------|---------|
| **家庭农场** | 单一地点，种植1-2种作物 | 5-20个设备 |
| **合作社** | 多个农户，各自管理 | 50-200个设备 |
| **农业企业** | 多个基地，统一管理 | 500-5000个设备 |
| **农业园区** | 多功能区（种植/养殖/加工） | 1000-10000个设备 |

### 1.2 核心需求

1. **层级管理**：基地 → 大棚/地块 → 区域 → 设备
2. **数据隔离**：不同基地数据相互独立
3. **权限控制**：管理员/技术员/工人 分级权限
4. **批量操作**：跨基地数据对比、批量控制
5. **资源复用**：设备模板、传感器配置、控制策略

## 2. 数据架构设计

### 2.1 实体关系模型

```
农场/基地 (Farm)
├── 区域 (Zone)
│   ├── 大棚/地块 (Greenhouse/Field)
│   │   ├── 传感器 (Sensor)
│   │   ├── 执行器 (Actuator)
│   │   └── 监控点 (Monitor)
│   └── 设备分组 (DeviceGroup)
├── 作物批次 (CropBatch)
│   ├── 种植计划
│   └── 生长周期
└── 用户权限 (UserPermission)
```

### 2.2 数据库表设计

```sql
-- 1. 农场/基地表
CREATE TABLE farms (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    code TEXT UNIQUE NOT NULL,        -- 基地编码（如：BJ-001）
    address TEXT,
    latitude REAL,
    longitude REAL,
    area REAL,                         -- 面积（亩）
    farm_type TEXT,                    -- 类型：greenhouse/field/mixed
    owner_id INTEGER,                  -- 负责人
    status TEXT DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 2. 区域表
CREATE TABLE zones (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    farm_id INTEGER NOT NULL,
    name TEXT NOT NULL,
    code TEXT NOT NULL,
    zone_type TEXT,                    -- 类型：greenhouse/field/warehouse
    area REAL,
    description TEXT,
    status TEXT DEFAULT 'active',
    FOREIGN KEY (farm_id) REFERENCES farms(id),
    UNIQUE(farm_id, code)
);

-- 3. 监控点表（大棚/地块内的具体位置）
CREATE TABLE monitor_points (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    zone_id INTEGER NOT NULL,
    name TEXT NOT NULL,
    code TEXT NOT NULL,
    point_type TEXT,                   -- 类型：raised_bed/soil_point/air_point
    position_x REAL,                   -- 坐标位置
    position_y REAL,
    description TEXT,
    FOREIGN KEY (zone_id) REFERENCES zones(id),
    UNIQUE(zone_id, code)
);

-- 4. 设备分组表
CREATE TABLE device_groups (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    farm_id INTEGER NOT NULL,
    name TEXT NOT NULL,
    group_type TEXT,                   -- 类型：sensor/actuator/mixed
    description TEXT,
    FOREIGN KEY (farm_id) REFERENCES farms(id)
);

-- 5. 扩展传感器表（添加基地关联）
ALTER TABLE sensors ADD COLUMN farm_id INTEGER;
ALTER TABLE sensors ADD COLUMN zone_id INTEGER;
ALTER TABLE sensors ADD COLUMN monitor_point_id INTEGER;
ALTER TABLE sensors ADD COLUMN device_group_id INTEGER;

-- 6. 扩展执行器表（添加基地关联）
ALTER TABLE actuators ADD COLUMN farm_id INTEGER;
ALTER TABLE actuators ADD COLUMN zone_id INTEGER;
ALTER TABLE actuators ADD COLUMN device_group_id INTEGER;

-- 7. 作物批次表
CREATE TABLE crop_batches (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    farm_id INTEGER NOT NULL,
    zone_id INTEGER NOT NULL,
    crop_type TEXT NOT NULL,           -- 作物类型
    variety TEXT,                      -- 品种
    batch_code TEXT UNIQUE NOT NULL,   -- 批次编码
    planting_date DATE,
    expected_harvest_date DATE,
    actual_harvest_date DATE,
    area REAL,                         -- 种植面积
    status TEXT DEFAULT 'growing',     -- 状态：planted/growing/harvested
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (farm_id) REFERENCES farms(id),
    FOREIGN KEY (zone_id) REFERENCES zones(id)
);

-- 8. 用户权限表
CREATE TABLE user_permissions (
    id INTEGER PRIMARY KEY SUPERKEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    farm_id INTEGER NOT NULL,
    role TEXT NOT NULL,                -- 角色：owner/admin/technician/worker
    permissions TEXT,                  -- JSON格式的权限列表
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (farm_id) REFERENCES farms(id),
    UNIQUE(user_id, farm_id)
);

-- 9. 设备模板表（用于快速创建设备）
CREATE TABLE device_templates (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    device_type TEXT NOT NULL,         -- sensor/actuator
    device_model TEXT,                 -- 型号
    default_config TEXT,               -- JSON格式的默认配置
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 10. 控制策略表（跨基地复用）
CREATE TABLE control_strategies (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    strategy_type TEXT,                -- 类型：irrigation/ventilation/lighting
    config TEXT NOT NULL,              -- JSON格式的策略配置
    is_template INTEGER DEFAULT 0,     -- 是否为模板
    created_by INTEGER,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 3. API接口设计

### 3.1 基地管理API

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/farms` | 获取基地列表 |
| GET | `/api/farms/[id]` | 获取基地详情 |
| POST | `/api/farms` | 创建基地 |
| PUT | `/api/farms/[id]` | 更新基地 |
| DELETE | `/api/farms/[id]` | 删除基地 |
| GET | `/api/farms/[id]/stats` | 获取基地统计数据 |

### 3.2 区域管理API

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/farms/[farmId]/zones` | 获取基地下的区域列表 |
| POST | `/api/farms/[farmId]/zones` | 创建区域 |
| PUT | `/api/zones/[id]` | 更新区域 |
| DELETE | `/api/zones/[id]` | 删除区域 |

### 3.3 设备管理API

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/devices` | 获取设备列表（支持按基地/区域筛选） |
| GET | `/api/devices/[id]` | 获取设备详情 |
| POST | `/api/devices` | 创建设备（关联基地/区域） |
| PUT | `/api/devices/[id]` | 更新设备 |
| DELETE | `/api/devices/[id]` | 删除设备 |
| POST | `/api/devices/batch` | 批量导入设备 |
| GET | `/api/devices/templates` | 获取设备模板 |

### 3.4 作物管理API

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/crop-batches` | 获取作物批次列表 |
| POST | `/api/crop-batches` | 创建作物批次 |
| PUT | `/api/crop-batches/[id]` | 更新作物批次 |
| GET | `/api/crop-batches/[id]/timeline` | 获取作物生长 timeline |

### 3.5 数据聚合API

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/analytics/farm/[id]` | 基地维度数据聚合 |
| GET | `/api/analytics/zone/[id]` | 区域维度数据聚合 |
| GET | `/api/analytics/compare` | 跨基地数据对比 |
| GET | `/api/analytics/crop/[id]` | 作物维度数据分析 |

## 4. 前端页面设计

### 4.1 页面结构

```
/dashboard                    # 总览仪表盘（可切换基地）
/farms                        # 基地列表
/farms/[id]                   # 基地详情
/farms/[id]/zones             # 区域管理
/farms/[id]/devices           # 设备管理
/farms/[id]/crops             # 作物管理
/farms/[id]/analytics         # 数据分析
/farms/[id]/settings          # 基地设置
```

### 4.2 核心页面功能

#### 4.2.1 总览仪表盘
- 基地切换选择器
- 各基地关键指标卡片
- 设备在线率统计
- 报警汇总
- 地图展示（可选）

#### 4.2.2 基地详情页
- 基地基本信息
- 区域平面图（可配置）
- 设备分布图
- 实时数据概览
- 快捷操作入口

#### 4.2.3 设备管理页
- 设备列表（支持树形展示：基地→区域→设备）
- 设备筛选（按基地/区域/类型/状态）
- 批量操作（批量添加、批量配置）
- 设备模板快速创建

## 6. 技术要点

### 6.1 数据隔离方案
- 每个API请求携带 `farm_id` 参数
- 数据库查询自动添加 `WHERE farm_id = ?` 条件
- 使用中间件统一处理权限校验

### 6.2 设备编号规则
```
[基地编码]-[区域编码]-[设备类型]-[序号]
示例：BJ-001-GH1-S-001（北京001号基地-1号大棚-传感器-001）
```

### 6.3 缓存策略
- 基地配置信息：Redis缓存，5分钟过期
- 设备列表：Redis缓存，1分钟过期
- 实时数据：不缓存，直接查询

### 6.4 性能优化
- 设备列表分页加载
- 区域树形结构懒加载
- 数据聚合使用数据库视图
- 历史数据按时间分表
