# 设备数据绑定方案

## 核心概念

### 设备网关（Gateway）
- 有独立IP的设备，可以是：
  - 单个WiFi传感器（自身就是网关）
  - 多传感器聚合器（LoRa/Zigbee网关）
  - 工业网关（RS485转WiFi）

### 设备节点（Node）
- 没有独立IP的传感器
- 通过网关上报数据
- 有唯一设备ID（MAC或序列号）

## 数据库设计

```sql
-- 设备网关表
CREATE TABLE gateways (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    farm_id INTEGER NOT NULL,
    zone_id INTEGER,
    name TEXT NOT NULL,
    gateway_type TEXT NOT NULL,  -- wifi_sensor/lorawan_gateway/serial_gateway
    ip_address TEXT,
    mac_address TEXT,
    protocol TEXT,               -- mqtt/http/lorawan
    status TEXT DEFAULT 'online',
    last_heartbeat TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (farm_id) REFERENCES farms(id)
);

-- 设备节点表（挂在网关下）
CREATE TABLE device_nodes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    gateway_id INTEGER NOT NULL,
    node_id TEXT NOT NULL,        -- 设备唯一ID（MAC/序列号）
    name TEXT NOT NULL,
    node_type TEXT NOT NULL,      -- sensor/actuator
    sensor_type TEXT,             -- temperature/humidity/light等
    location TEXT,
    config TEXT,                  -- JSON配置
    status TEXT DEFAULT 'online',
    last_update TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (gateway_id) REFERENCES gateways(id),
    UNIQUE(gateway_id, node_id)
);

-- 设备数据表（统一存储所有数据）
CREATE TABLE device_data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    gateway_id INTEGER NOT NULL,
    node_id TEXT NOT NULL,
    sensor_type TEXT NOT NULL,
    value REAL NOT NULL,
    unit TEXT,
    quality INTEGER DEFAULT 100,  -- 数据质量 0-100
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (gateway_id) REFERENCES gateways(id)
);
```

## 数据上报流程

### 场景1：WiFi直连传感器
```
POST /api/device/report
{
    "gateway_ip": "192.168.1.101",
    "gateway_type": "wifi_sensor",
    "mac": "AA:BB:CC:DD:EE:FF",
    "data": [
        {"type": "temperature", "value": 25.5, "unit": "°C"}
    ]
}
```

### 场景2：网关聚合上报
```
POST /api/device/report
{
    "gateway_ip": "192.168.1.100",
    "gateway_type": "lorawan_gateway",
    "mac": "11:22:33:44:55:66",
    "nodes": [
        {
            "node_id": "sensor_001",
            "type": "temperature",
            "value": 24.5,
            "unit": "°C"
        },
        {
            "node_id": "sensor_002", 
            "type": "humidity",
            "value": 65.0,
            "unit": "%"
        }
    ]
}
```

## 自动绑定逻辑

1. **首次上报**：自动创建网关和设备节点
2. **IP/MAC匹配**：后续上报自动关联到已有设备
3. **基地绑定**：通过网关的farm_id关联到基地
4. **位置更新**：支持手动调整设备位置

## 优势

1. **灵活扩展**：支持任意数量的传感器
2. **自动发现**：新设备首次上报自动注册
3. **数据隔离**：通过farm_id实现多基地数据隔离
4. **兼容性好**：支持各种物联网协议和设备
